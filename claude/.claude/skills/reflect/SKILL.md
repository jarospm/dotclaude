# Session Reflection

Reflect on the work done in this session. Review the changes made and provide honest, thoughtful answers to these questions.

## Instructions

1. Review the work done during this session — code changes, plans, research, documents
2. Answer each question below with specifics — reference actual code, files, and decisions
3. Be candid about uncertainties and rejected alternatives
4. Keep responses concise but substantive

## Reflection Questions

### Decisions

- **What was the hardest decision you made here?**
  Explain the tension or trade-off that made it difficult.

- **What alternatives did you reject, and why?**
  List specific approaches you considered but didn't use.

### Confidence

- **What are you least confident about?**
  Flag anything that might need review, testing, or a second opinion.

- **What assumptions did you make?**
  Note any implicit assumptions about the codebase, requirements, or user intent.

### Follow-up

- **What would you do differently with more time?**
  Identify improvements you'd make if scope allowed.

- **What should the user know before proceeding?**
  Surface any risks, gaps, dependencies, or required next steps.
